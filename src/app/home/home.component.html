Hemidall

El mundo corporativo esta en el proceso de trascender de las practicas y el trabajo manual a la automatización, pero, aunque algunas empresas se nieguen a seguir con el movimiento, eso no detiene a las grandes empresas de TI seguir adelante y llevar estos procesos a un nivel más allá de la simple automatización. Ese nivel es la autonomía.

La autonomía de estos procesos, especialmente procesos de TI como la Integración Continua, la cual incluye comúnmente las pruebas de software, y la Entrega Continua, son una de las mayores claves para el éxito de estas empresas, ya que están tratando de ganar la carrera contra sus competidores al ser los primeros en desplegar nuevas tecnologías y ser capaces de arreglar fallas con sus sistemas para mantener una mayor estabilidad y tiempo de disponibilidad de sus aplicaciones o servicios de TI.

Objetivos Heminadall

El primordial objetivo de este proyecto es el desarrollo de una herramienta la cual permita la creación y la calidad de las pruebas de software de una manera más rápida y eficaz, utilizando herramientas y tecnologías emergentes como el Machine Learning y la Inteligencia Artificial se pretende crear una herramienta que le permita al usuario crear pruebas a través de una estimación de los objetos que pueda contener una aplicación web.



Alcance Heminadll

Usando Reconocimiento Óptico de Caracteres la herramienta pretende buscar un elemento web relacionado a la palabra clave o tag que el usuario le suministre a la aplicación a través de modelos entrenados de Machine Learning, los cuales serán entrenados con una amplia variedad de iconos recopilados a través de diversas aplicaciones web para asegurar su correcto funcionamiento y su viabilidad como una herramienta independiente como suplemento para la creación más rápida y de mejor calidad de una aplicación web, todo esto a través de una API, la cual podría ser integrada a cualquier aplicación y accesada con un token de seguridad.

La aplicación no será un framework para pruebas automatizadas sino una herramienta más la cual podrá ser usada en cualquier otro framework o a través de una interfaz gráfica la cual será proporcionada a forma de demo por la aplicación.
